# UI改善要求定義書

## 概要
サイドバー、ウィンドウコントロール、背景パターンのUI改善を行う。

## 変更要求

### 1. サイドバーの簡素化
- **問題**: 枠が多すぎる、アイコンがカラフルすぎる
- **要求**:
  - 余分なボーダーを削除
  - アイコンを単色に統一
  - ロゴを「ILI」から「ILIS」に変更
  - VSCode風のシンプルなホバーエフェクト

### 2. ウィンドウコントロールの改善
- **問題**: ボタンにborderがある
- **要求**:
  - borderを削除
  - VSCode風のホバーエフェクト（背景色のみ変化）
  - 閉じるボタンは赤色背景

### 3. 背景パターンの最適化
- **問題**: SVGが重い、円が重なる
- **要求**:
  - SVGからPNGに変更
  - ドーナツ型（中央が空洞）のパターン
  - 3つの円が重ならない配置
  - すべて薄い青色で統一

## 受け入れ条件

1. サイドバー
   - [ ] ボーダーが最小限になっている
   - [ ] アイコンが単色で表示される
   - [ ] ロゴが「ILIS」と表示される
   - [ ] ホバー時に背景色のみが変化する

2. ウィンドウコントロール
   - [ ] ボタンにborderがない
   - [ ] ホバー時に背景色が変化する
   - [ ] 閉じるボタンのホバーが赤色

3. 背景パターン
   - [ ] PNG形式で表示される
   - [ ] 3つのパターンが重ならない
   - [ ] パフォーマンスが改善される

## 制約事項
- 既存のPNGファイルは保持する（別名で新規作成）
- Canva MCPのデザインガイドラインを参考にする
