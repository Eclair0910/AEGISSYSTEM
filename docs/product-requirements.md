# プロダクト要求定義書

## 1. プロダクトビジョンと目的

### ビジョン
AEGISSystemは、近未来SF的なデザインを特徴とする、複数の機能を統合したデスクトップアプリケーションです。直感的なUIと視覚的に魅力的なデザインにより、ユーザーに新しいデスクトップ体験を提供します。

### 目的
- システム情報やパフォーマンスをリアルタイムで監視できる環境を提供
- 近未来的なUIデザインによる視覚的な満足感の提供
- 複数の機能を一つのアプリケーションに統合し、利便性を向上

## 2. ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
- システム監視に関心があるユーザー
- 近未来的・SF的なデザインを好むユーザー
- デスクトップアプリケーションを日常的に利用するユーザー

### 課題・ニーズ
| 課題 | ニーズ |
|------|--------|
| 既存のシステム監視ツールはデザインが地味 | 視覚的に魅力的なモニタリングツール |
| 複数のツールを切り替える煩わしさ | 統合されたマルチ機能アプリケーション |
| OSネイティブのウィンドウデザインが好みでない | カスタムウィンドウデザイン |

## 3. 主要な機能一覧

### コア機能

| 機能名 | 説明 | 優先度 |
|--------|------|--------|
| スプラッシュ画面 | 起動時に丸形のローディングアニメーションを表示 | 高 |
| Mainページ | アプリケーションのメインダッシュボード | 高 |
| Performanceページ | CPU、メモリ使用量などをリアルタイム監視 | 高 |
| カスタムウィンドウコントロール | 独自の最小化・最大化・閉じるボタン | 高 |

### UI/UX機能

| 機能名 | 説明 | 優先度 |
|--------|------|--------|
| 近未来SFデザイン | サイバーパンク風のUIテーマ | 高 |
| 歯車回転背景 | 背景に回転する円形・歯車模様のアニメーション | 中 |
| フレームレスウィンドウ | OSネイティブのウィンドウ枠を非表示 | 高 |

## 4. 成功の定義

### 定量的指標
- アプリケーションの起動時間：5秒以内
- メモリ使用量：200MB以下
- CPU使用率：アイドル時5%以下

### 定性的指標
- ユーザーがUIデザインに満足する
- スムーズなアニメーション表示
- 直感的なナビゲーション

## 5. ビジネス要件

### 技術要件
- **フレームワーク**: Electron
- **フロントエンド**: React + TypeScript
- **バックエンド**: Node.js
- **スタイリング**: SCSS
- **構造**: HTML/CSSを分離

### プラットフォーム要件
- Windows 10/11対応
- macOS対応（オプション）
- Linux対応（オプション）

## 6. ユーザーストーリー

### US-001: スプラッシュ画面の表示
**As a** ユーザー
**I want** アプリ起動時に近未来的なローディング画面を見たい
**So that** アプリの起動を待つ間も視覚的な楽しみを得られる

**受け入れ条件:**
- [ ] 丸形のローディングインジケーターが表示される
- [ ] 弧を描くように0%から100%までプログレスが表示される
- [ ] ローディング完了後、自動的にMainページへ遷移する

### US-002: Mainページの表示
**As a** ユーザー
**I want** アプリのメインダッシュボードを見たい
**So that** 各機能にアクセスできる

**受け入れ条件:**
- [ ] 近未来SF的なデザインで表示される
- [ ] 背景に歯車模様が回転している
- [ ] 各機能へのナビゲーションが表示される

### US-003: パフォーマンス監視
**As a** ユーザー
**I want** コンピュータのパフォーマンスをリアルタイムで確認したい
**So that** システムの状態を把握できる

**受け入れ条件:**
- [ ] CPU使用率が表示される
- [ ] メモリ使用量が表示される
- [ ] 情報がリアルタイムで更新される

### US-004: ウィンドウコントロール
**As a** ユーザー
**I want** アプリ独自のウィンドウコントロールを使いたい
**So that** 統一されたデザイン体験を得られる

**受け入れ条件:**
- [ ] OSネイティブのウィンドウ枠が非表示
- [ ] 右上に最小化ボタンが表示される
- [ ] 右上に最大化ボタンが表示される
- [ ] 右上に閉じるボタンが表示される
- [ ] 各ボタンが正しく機能する

## 7. 受け入れ条件（全体）

### 必須条件
- [ ] Electron + React + TypeScript + Node.jsで構築されている
- [ ] SCSSでスタイリングされている
- [ ] HTML/CSSが分離されている
- [ ] スプラッシュ画面が実装されている
- [ ] Mainページが実装されている
- [ ] Performanceページが実装されている
- [ ] カスタムウィンドウコントロールが実装されている
- [ ] 近未来SF的なUIデザインが適用されている
- [ ] 背景に歯車模様のアニメーションがある

### 品質条件
- [ ] TypeScriptの型チェックがパスする
- [ ] ESLintのチェックがパスする
- [ ] アプリケーションがクラッシュしない

## 8. 機能要件

### FR-001: スプラッシュ画面
| 項目 | 内容 |
|------|------|
| ID | FR-001 |
| 機能名 | スプラッシュ画面 |
| 説明 | 起動時にローディング画面を表示 |
| 入力 | なし |
| 処理 | 円形プログレスバーを0%→100%までアニメーション |
| 出力 | 完了後Mainページへ遷移 |

### FR-002: Mainページ
| 項目 | 内容 |
|------|------|
| ID | FR-002 |
| 機能名 | Mainページ |
| 説明 | アプリケーションのメインダッシュボード |
| 入力 | なし |
| 処理 | ナビゲーションメニューの表示、背景アニメーションの描画 |
| 出力 | 各機能へのナビゲーション |

### FR-003: Performanceページ
| 項目 | 内容 |
|------|------|
| ID | FR-003 |
| 機能名 | Performanceページ |
| 説明 | システムパフォーマンスのリアルタイム監視 |
| 入力 | システム情報（OS API経由） |
| 処理 | CPU、メモリ情報の取得・表示・更新 |
| 出力 | リアルタイムグラフ/数値表示 |

### FR-004: カスタムウィンドウコントロール
| 項目 | 内容 |
|------|------|
| ID | FR-004 |
| 機能名 | カスタムウィンドウコントロール |
| 説明 | 独自のウィンドウ操作ボタン |
| 入力 | ユーザーのクリック操作 |
| 処理 | Electron APIを通じてウィンドウを操作 |
| 出力 | 最小化/最大化/閉じる動作 |

## 9. 非機能要件

### NFR-001: パフォーマンス
| 項目 | 要件 |
|------|------|
| 起動時間 | 5秒以内 |
| アイドル時CPU使用率 | 5%以下 |
| メモリ使用量 | 200MB以下 |
| アニメーションFPS | 30fps以上 |

### NFR-002: ユーザビリティ
| 項目 | 要件 |
|------|------|
| レスポンシブ | ウィンドウリサイズに対応 |
| 視認性 | 重要な情報が明確に表示される |
| 操作性 | 直感的なナビゲーション |

### NFR-003: 保守性
| 項目 | 要件 |
|------|------|
| コード品質 | TypeScript厳格モード |
| スタイル管理 | SCSS変数による一元管理 |
| コンポーネント設計 | 再利用可能なコンポーネント構造 |

### NFR-004: セキュリティ
| 項目 | 要件 |
|------|------|
| コンテキスト分離 | contextIsolation: true |
| Node統合 | nodeIntegration: false (preload経由でアクセス) |
| XSS対策 | ユーザー入力のサニタイズ |
| **インターネットアクセス** | **禁止（ローカル専用アプリ）** |

### NFR-005: インターネットアクセス禁止
| 項目 | 要件 |
|------|------|
| 外部通信 | 一切のインターネット通信を禁止 |
| 理由 | システム情報を扱うため、情報漏洩リスクを排除 |
| 外部API | fetch, axios, WebSocket等の使用禁止 |
| 外部リソース | CDN、外部フォント、外部スクリプト読み込み禁止 |
| テレメトリ | 使用状況・エラー報告の外部送信禁止 |
